## 第6章 路由器、交换机及其操作系统介绍

*   **路由器与交换机的基本概念**：
    *   路由器（Router）是构建各种规模企业网络的核心设备。
    *   路由器主要工作在网络层（第三层），根据 IP 地址信息进行报文转发，实现不同网络间的互联互通。
    *   交换机（Switch）主要工作在数据链路层（第二层），根据 MAC 地址信息进行数据帧交换，实现局域网内部的高速数据转发。
*   **路由器与交换机的区别**：
    *   路由器的核心功能是进行路由选择，决定最佳路径，连接不同网络。
    *   交换机的核心功能是进行数据交换，提高局域网的带宽利用率，连接同一网络。
    *   路由器可以连接不同类型的网络，交换机主要用于连接同一类型的网络。
*   **H3C Comware**：
    *   Comware 是 H3C 公司开发的网络设备操作系统，是 H3C 产品线的核心软件平台。
    *   Comware 负责管理和控制硬件设备，提供各种网络功能，如路由、交换、安全等。
    *   Comware 的特点：
        *   支持 IPv4 和 IPv6 双协议栈。
        *   支持多核 CPU，增强设备的处理能力。
        *   同时提供路由和交换功能。
        *   高可靠性和弹性扩展。
        *   灵活的裁减和定制功能。

## 第7章 命令行操作基础

*   **访问网络设备命令行接口的方法**：
    *   Console 口本地访问：通过 Console 线缆直接连接到设备的 Console 端口，无需网络。
    *   AUX 口远程访问：通过 Modem 和电话线进行远程访问。
    *   Telnet 终端访问：通过 Telnet 协议进行远程登录。
    *   SSH 终端访问：通过 SSH 协议进行加密的远程登录，更安全。
    *   异步串口访问：通过异步串口进行访问
*   **命令行使用入门**：
    *   用户视图：设备启动后的缺省视图。
    *   系统视图：配置系统全局通用参数的视图。
    *   接口视图：配置接口参数的视图。
    *   路由协议视图：配置路由协议参数的视图。
    *   用户线视图：配置登陆设备的各个用户属性的视图。
*   **常用命令**：
    *   `display`: 显示系统配置信息和维护信息。
    *   `interface`: 进入接口视图。
    *   `ip address`: 配置 IP 地址。
    *   `quit`: 退出当前视图。
    *   `system-view`: 进入系统视图。
*   **命令行帮助特性**：
    *   在命令行中输入 `?` 可以查看当前视图下可用的命令列表。
    *   输入命令的一部分后按 `Tab` 键可以自动补全命令。
    *   用户角色：
        *   `network-admin`拥有最高的权限，可以操作系统所有的功能和资源。
        *   `network-operator`可以执行有限的功能和资源相关的`display`命令。

## 第8章 网络设备文件管理

*   **网络设备文件系统介绍**：
    *   定义：设备以文件的方式对运行所需的数据进行存储
    *   存储方式：ROM, Flash, RAM
        *   `ROM`：只读存储器，用于存放BootROM程序
        *   `Flash`：用于存储操作系统文件、配置文件等
        *   `RAM`：随机访问存储器，用于暂存系统运行的数据

*   **文件系统的操作**：
    *   目录操作：创建/删除目录, 显示当前工作目录以及显示指定目录下的文件或目录等
        *   `mkdir directory`
        *   `rmdir directory`
        *   `pwd`
        *   `dir [/all ] [ file-url]`
        *   `cd directory`
    *   文件操作：删除文件, 恢复删除的文件, 彻底删除文件, 显示文件的内容, 重命名文件, 拷贝文件, 移动文件, 显示指定的文件的信息等
        *   `delete [/unreserved] file-url`
        *   `undelete file-url`
        *   `reset recycle-bin [ file-url ] [ /force]`
        *   `more file-url`
        *   `rename fileurl-source fileurl-dest`
        *   `copy fileurl-source fileurl-dest`
        *   `move fileurl-source fileurl-dest`
        *   `dir [/all ] [ file-url]`
    *   存储设备操作：恢复存储设备的空间, 格式化存储设备等
        *   `fixdisk device`
        *   `format device`
    *   设置文件系统的提示方式
        *   `file prompt { alert | quiet }`

*   **FTP服务的配置与使用**：
    *   启用FTP服务
        *   `ftp server enable`
    *   创建用户
        *   `local-user username`
    *   设置服务类型及登录密码
        *   `[H3C-luser-manage-xxx] service-type ftp`
        *   `[H3C-luser-manage-xxx] password { hash | simple } password`
*   **TFTP的配置与使用**：
    *   TFTP 客户端：通过tftp命令进行文件下载和上传
        *   `tftp tftp-server { get | put | sget } source-filename [ destination-filename ] [ source { interface interface-type interface-number| ip source-ip-address } ]`

## 第9章 系统调试

*   **ping、tracert命令检查网络连通性**：
    *   Ping命令：
        *   基于ICMP协议，测试网络中两台设备间的连通性。
        *   `ping ip_address`
    *   Tracert命令：
        *   跟踪数据包经过的路由，用于诊断网络故障。
        *   `tracert ip_address`
*   **debug等命令进行网络系统基本调试**：
    *   `terminal debugging`命令的作用是打开调试信息的屏幕输出开关
    *   `debugging`命令用于打开协议调试开关
